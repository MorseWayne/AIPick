# 小红书购买意向分析与商品推荐系统 (AIPick)

AIPick 是一个基于大语言模型（LLM）与小红书 MCP (Model Context Protocol) 服务的智能商品选购助手。它通过分析用户的自然语言购买需求，利用小红书真实的图文笔记和海量评论做深度多维度评估（如提取优缺点、过滤水军、计算性价比与好中差评），最终生成极具参考价值的结构化商品推荐榜单。

## 系统特性 (Features)

1. **自然语言意图提取与多轮交互追问**：利用 LLM 作为实体抽取器结构化提取“商品类目”、“预算”及“核心诉求”。若信息不足，系统将基于固化的状态机机制主动追问用户。
2. **多源真实数据支撑**：通过对接底层的 `xiaohongshu-mcp` HTTP 服务，自动搜索热点笔记、提取测评正文及下方高赞评论数据。
3. **结构化深度评估**：通过 Pydantic 结合大模型的 Structured Outputs 特性，稳定输出结构化的评价报告。
4. **多维度打分看板**：自动统计商品的优缺点、估算好差评比率、计算性价比并提供全局的评分卡片。

## 运行环境准备 (Setup & Installation)

推荐使用现代的 Python 包和环境管理器工具 **`uv`** 来运行本项目，以实现极速的依赖隔离安装。

### 1. 克隆代码并进入目录

```bash
cd /your/path/to/AIPick
```

### 2. 使用 `uv` 构建虚拟环境并安装依赖

```bash
# 使用 uv 创建虚拟环境并安装核心跨平台依赖
uv venv
uv pip install -r requirements.txt
```

### 3. 配置环境变量

在你的项目根目录下复制 `.env.example` 并重命名为 `.env`：

```bash
cp .env.example .env
```

随后编辑 `.env` 文件，填入你自己所需要使用的 LLM API 密钥以及对应的服务地址以驱动核心的大模型 Agent。

```ini
# .env 文件示例
OPENAI_API_KEY=sk-xxxxxx...
OPENAI_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o

# 若有变动可配置 MCP 服务器地址
XHS_MCP_URL=http://10.10.131.118:18060/mcp
```

## 运行与使用 (Usage)

在终端中使用 `uv run` 启动入口文件：

```bash
uv run python main.py
```

## 常见问题与后续扩展

- **MCP 解析出错**：由于小红书的组件和 MCP 服务器返回值经常处于调整中。当你发现命令行报解析 JSON 相关结构体失败的错误时，请前往 `src/agent.py` 文件和 `src/xhs_mcp_client.py` 中寻找搜索和拉取详情返回值的反序列化相关逻辑进行调整适配。
- **登录鉴权重定向**：本项目暂未集成自动拉取 Base64 登录二维码的能力，如果遇到需要登陆，请确保当前部署 MCP 服务器的节点保持 Cookie 活性。
- **Token 限流策略**：大范围检索评论（例如一次抓取100条长评）会急剧增加发送给 LLM 的上下文窗口，推荐使用大视窗模型（如智谱GLM4、GPT-4o 或 Claude 3.5 Sonnet 等）。
